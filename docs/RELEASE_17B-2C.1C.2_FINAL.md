# Release 17B-2C.1C.2 FINAL

Финальная фиксация фактического production-состояния платёжного ядра ЮKassa,
миграции `0002_payment_core_yookassa`, изменений клиентских и CRM-страниц,
Telegram-уведомлений, очистки каталога и архитектурной документации.

## Безопасный режим

- Реальные реквизиты не входят в Git и release-архив.
- `YOOKASSA_SECRET_KEY` в `.env.example` остаётся пустым.
- `YOOKASSA_TEST_MODE=true` является документированным значением по умолчанию.
- При отсутствии Shop ID/Secret Key онлайн-оплата должна оставаться выключенной.
- Проверочные сценарии не обращаются к сети ЮKassa.

## Состояние базы

- Drizzle migration: `0002_payment_core_yookassa`.
- Каноническая схема production: 36 таблиц, 460 колонок.
- Добавлены `payment_events`, расширенные статусы и поля платёжной попытки.

## Git

Финализатор создаёт локальные commits и annotated tag. Push в `origin` по умолчанию
не выполняется и требует отдельного явного запуска с `PUSH_TO_ORIGIN=1` после
проверки адреса репозитория.
