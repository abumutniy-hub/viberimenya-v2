# RELEASE 17B-2C.3.6.6 — Customer Auth, Bouquet Approval and Order Details

Базовый production commit: `4f3dd75708be513a681a7e785afc05192b19a8d3`.

## Исправлено

- Telegram-регистрация нового покупателя по deep link, QR, ручному коду и кнопке передачи собственного номера.
- Вход уже подключённого покупателя подтверждается сразу, без ошибочной второй кнопки.
- Повторная обработка состояний `confirmed` и `consumed` идемпотентна.
- Фото готового букета отправляется в Telegram непосредственно файлом из локального `/uploads`.
- При ошибке изображения покупатель всё равно получает текст и кнопки согласования.
- Неудавшееся уведомление `bouquet_approval_requested` ставится на повторную доставку.
- Фото, статус согласования, дата, адрес и интервал доставки добавлены в личный кабинет.
- Страница отслеживания получает интервал через `delivery_intervals`.
- Большая мобильная панель checkout исправлена устранением наследования общего footer padding.
- Мобильная страница заказа стала компактнее.
- В CRM и юридические страницы добавлены банковские реквизиты магазина.

## Безопасность

- SMS не добавляются.
- Схема БД и миграции не меняются.
- Переданный Telegram-контакт обязан принадлежать отправителю и совпадать с номером на сайте.
- Локальный путь фотографии проверяется от path traversal.
- Полный доступ к профилю создаётся только после Telegram-подтверждения.


Дополнение 3.6.6: служебные API-скрипты используют `createDb` из `@viberimenya/db`; прямой импорт `postgres` из `apps/api` удалён.
