# RELEASE 17B-2C.3.6.7 — CUSTOMER AUTH SAME-BROWSER FIX

Исправляет Telegram-вход на планшетах и телефонах, когда пользователь создаёт запрос в Яндекс.Браузере, переходит в Telegram и возвращается обратно.

Причина: все запросы использовали одну глобальную HttpOnly cookie. Новый запрос или соседняя вкладка перезаписывали её, после чего исходная вкладка ошибочно получала `invalid_browser`.

Изменения:

- отдельная HttpOnly cookie для каждого pairing requestId;
- cookie ограничена API-путём своего запроса;
- параллельные вкладки больше не перезаписывают друг друга;
- новый запрос не отменяет предыдущий активный запрос;
- pairing intent сохраняется в `sessionStorage` и восстанавливается после возврата из Telegram;
- старый глобальный формат cookie поддерживается на переходный период и очищается после входа;
- схема базы и migrations не меняются;
- SMS не добавляются.
