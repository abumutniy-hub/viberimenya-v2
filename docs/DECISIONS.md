# Технические решения

## Архитектура

Проект строится как SaaS-ready monorepo.

На старте используется один магазин, но все ключевые сущности проектируются с `shop_id`.

## Стек

- Node.js 22
- pnpm
- PostgreSQL 16
- Nginx
- PM2
- TypeScript
- Next.js для web
- Fastify для API
- Drizzle ORM для базы
- grammY для Telegram-бота

## Почему не полноценный SaaS сразу

Полный SaaS увеличит сложность и отложит запуск. Поэтому делаем один магазин, но с архитектурой под масштабирование.
