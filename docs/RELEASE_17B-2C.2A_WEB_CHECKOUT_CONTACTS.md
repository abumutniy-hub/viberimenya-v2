# Release 17B-2C.2A — Web Checkout: покупатель и получатель

## Назначение

Первый этап отдельного пошагового оформления заказа на сайте. Страница
`/checkout` работает с тем же серверным checkout draft, который используется
Telegram-ботом.

## Реализовано

- отдельная mobile-first страница `/checkout`;
- обязательный вход в клиентский профиль;
- обязательная единая Telegram-идентификация для межканального черновика;
- автозаполнение имени, подтверждённого телефона и email из профиля;
- подтверждённый телефон не редактируется внутри checkout;
- отдельный получатель или режим «Получатель — я»;
- флаги «Это сюрприз» и «Не звонить получателю»;
- предпочтительный способ связи с покупателем;
- клиентская и серверная проверка имени, телефона и email;
- нормализация российских телефонных номеров;
- autosave в server draft с revision control;
- обработка конфликта между сайтом и Telegram;
- section-level merge при подтверждённом конфликте;
- сохранение более позднего Telegram progress без отката назад;
- переход в реальный раздел доставки существующего checkout;
- восстановление контактов из общего draft в корзине;
- вход в пошаговый checkout из корзины;
- `/checkout` исключён из индексации.

## Безопасность

- PII не сохраняется в localStorage новой страницы;
- server draft имеет TTL 24 часа;
- API требует активную customer session;
- черновик привязан к единой Telegram identity;
- optimistic revision блокирует устаревшую вкладку;
- operation ID защищает повторные запросы;
- подтверждённый телефон берётся из профиля;
- server validation является источником истины;
- schema и migrations не меняются.

## Не входит в 2A

- зоны, даты, интервалы и адрес — этап 2B;
- промокод, бонусы, способ оплаты и финальный review — этап 2C;
- новая финализация заказа не создаётся: используется уже проверенное
  атомарное ядро.
