# RELEASE 17B-2C.3.3 — Launch readiness unknown error fix

## Назначение

Повторная безопасная установка центра готовности к запуску после автоматического отката `17B-2C.3.2`.

## Исправления

- общий API error handler больше не обращается к значению типа `unknown` напрямую;
- `statusCode` читается только после проверки, что ошибка является объектом и содержит числовое поле;
- malformed JSON по-прежнему возвращает `400 Invalid request`, а не `500`;
- валидный запрос контрольного заказа без CRM-сессии возвращает `401` или `403`;
- сохранены strict optional-поля, контрольный заказ и все launch-readiness проверки.

## База данных

Схема, таблицы и migrations не изменяются.
