# Release 17B-2C.2B — Web Checkout: доставка и адрес

## Назначение

Второй этап отдельного пошагового оформления заказа. Страница
`/checkout/delivery` использует тот же server checkout draft, что сайт и
Telegram-бот.

## Реализовано

- отдельный шаг доставки `/checkout/delivery`;
- доставка курьером и самовывоз;
- выбор зоны и обычного/срочного тарифа;
- дата от текущего дня до 180 дней вперёд;
- выбор активного интервала;
- сохранённые адреса клиента;
- автоподсказки российских адресов через server-side DaData proxy;
- debounce 350 мс и запрос только от трёх символов;
- ручное подтверждение адреса при недоступном провайдере;
- квартира/офис, подъезд, этаж, домофон и комментарий курьеру;
- режим «Частный дом / квартиры нет»;
- ФИАС, КЛАДР, индекс, регион, город, улица, дом, координаты и качество
  геокодирования в общем JSON draft;
- восстановление нового адреса и даты в существующей корзине;
- передача структурированного адреса из Telegram при финализации;
- полный адрес и детали доступа в заказе;
- структурированные данные в `orders.metadata.delivery.addressDetails`;
- autosave с revision/operationId и сохранением более позднего progress.

## Безопасность

- DaData token находится только в `.env` API-сервера;
- браузер обращается только к собственному authenticated endpoint;
- endpoint ограничен rate limit 40 запросов в минуту на клиента/IP;
- внешний запрос имеет тайм-аут;
- в browser response передаётся только необходимый набор полей;
- при ошибке провайдера checkout не блокируется: доступен ручной ввод;
- адрес проходит клиентскую и серверную проверку;
- schema и migrations не меняются;
- существующий атомарный order core остаётся единственным способом создать
  заказ.

## Переменные окружения

```env
DADATA_API_TOKEN=
DADATA_LOCATION_BOOST_KLADR_ID=
DADATA_REQUEST_TIMEOUT_MS=4000
```

Без `DADATA_API_TOKEN` страница работает в ручном режиме.

## Не входит в 2B

- автоматическая проверка попадания координат в polygon зоны;
- ограничения пропускной способности интервалов;
- шаг оплаты, промокода, бонусов и финального review — этап 2C.
