# RELEASE 17B-2C.3.5 — Guest Checkout + Telegram Auth Restore / E2E Runner Fix

## Цель

Вернуть исходную коммерческую логику магазина: покупатель оформляет заказ без обязательной регистрации, SMS и предварительной привязки Telegram. Существующая безопасная авторизация через `@viberimenya_bot` остаётся единственным способом получить полный доступ к личному кабинету.

## Покупательский путь

`Корзина → Контакты → Доставка → Итог → Заказ → Подключение Telegram`

- Гостю выдаётся случайная HttpOnly-cookie на 24 часа.
- Корзина и checkout-draft изолированы в отрицательном bigint scope и не пересекаются с Telegram ID.
- Во время создания заказа клиент находится или создаётся по нормализованному телефону.
- Гостю выдаётся только секретная ссылка конкретного заказа.
- Полная customer-session без Telegram-подтверждения не создаётся.
- Если Telegram не подключён, после заказа показываются код и кнопка `@viberimenya_bot`.
- Бонусы и история аккаунта доступны только после входа через Telegram.

## Безопасность

- Cookie: `HttpOnly`, `SameSite=Lax`, `Secure` в production.
- Случайный токен: 256 бит.
- Гостевой scope стабилен только в пределах cookie и не раскрывается браузеру.
- Ввод чужого номера не даёт доступ к старым заказам, адресам и бонусам.
- Данные существующего профиля не перезаписываются гостем.
- Привязка Telegram по коду по-прежнему требует подтверждения номера через Telegram contact.

## База данных

Схема и migrations не меняются. Используются существующие таблицы корзины, checkout-draft, клиентов, заказов и токенов привязки.


## Исправление установщика

Web E2E-файлы запускаются через `tsx` из workspace API, где бинарник действительно установлен. Код гостевого checkout не изменён относительно подготовленного 17B-2C.3.4.
