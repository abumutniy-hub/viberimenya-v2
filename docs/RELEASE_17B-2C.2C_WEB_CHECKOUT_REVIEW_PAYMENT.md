# RELEASE 17B-2C.2C — Web Checkout Review & Payment

## Цель

Завершить единую web-цепочку оформления заказа: контакты → доставка → проверка, скидки и оплата → созданный заказ.

## Реализовано

- отдельная страница `/checkout/review`;
- серверная загрузка единой корзины и checkout draft;
- итоговая карточка товаров, покупателя, получателя и доставки;
- открытка, сюрприз, запрет звонка и комментарий;
- промокод и бонусы через server quote;
- способы оплаты только из актуальных настроек магазина;
- онлайн-карта и СБП отображаются только при включённой и настроенной ЮKassa;
- обязательное согласие с обработкой данных и офертой;
- финальная повторная проверка цены, остатков, доставки, промокода и бонусов;
- атомарный `POST /api/public/orders` с `clientRequestId`;
- защита от повторного заказа;
- очистка корзины и draft только после успешного ответа;
- переход на страницу отслеживания с понятным сообщением;
- старая параллельная форма заказа в `/cart` отключена.

## Оплата

Для `online_card` и `sbp` заказ сначала создаётся в статусе `new`. После подтверждения менеджером на странице отслеживания становится доступно создание платёжной ссылки ЮKassa. Это соответствует действующему payment core и не позволяет оплатить неподтверждённый или недоступный заказ.

## База данных

Схема и миграции не изменяются.
