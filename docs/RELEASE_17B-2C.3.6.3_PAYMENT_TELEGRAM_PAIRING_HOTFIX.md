# RELEASE 17B-2C.3.6.3 — PAYMENT + TELEGRAM PAIRING HOTFIX

Исправлены два блокера контрольного заказа:

- `transfer_after_confirm` остаётся доступным при отключённой или ненастроенной ЮKassa, даже когда в настройках включена оплата при получении;
- WEB и API используют одно правило доступности способа оплаты;
- резервный Telegram-код проверяется до сценария Telegram-checkout;
- deep-link поддерживает текущий SHA-256 token и совместимый legacy token;
- ручной код поддерживает оба формата метаданных;
- если deep-link не распознан, активный запрос можно восстановить безопасной передачей собственного номера Telegram.

Схема базы и migrations не изменяются. SMS не добавляются.
