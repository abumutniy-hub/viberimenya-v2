# RELEASE 17B-2C.3.6.13.1 — Bouquet approval endpoint + photo viewer

## Исправлено

- Явная типизация динамических параметров `jsonb_build_object` через `::text` устраняет PostgreSQL `42P18` при решениях «Одобряю», «Всё подходит» и «Нужна правка».
- Тип события и nullable-комментарий уведомления также имеют явный `text` cast.
- Добавлен полноэкранный просмотр фотографии букета в профиле, «Моих заказах» и странице отслеживания.
- Viewer поддерживает кнопки увеличения, двойное нажатие, pinch-жест, сброс масштаба, Escape и закрытие по фону.

## Не изменяется

- Telegram browser pairing и JSONB text-cast авторизации.
- Оплата и checkout.
- Схема базы и миграции.
- Права uploads и Nginx-конфигурация.

## Исправление установщика 3.6.13.1

- Изолированный Git preflight сначала фиксирует точный baseline, а затем накладывает payload.
- Существующие пробелы и пустые строки в неизменённых production-файлах больше не считаются ошибкой релиза.
- Проверка `git diff --check` применяется только к изменениям этого релиза.
