# ВЫБЕРИ МЕНЯ v2 — инструкции для Claude Code

## Кто пользователь
Пользователь не программист. Все действия нужно объяснять простым языком. Не делать хаотичных изменений. Работать строго по одному блоку за раз.

## Проект
Интернет-магазин цветов "ВЫБЕРИ МЕНЯ v2".

Монорепозиторий:
- apps/web — Next.js сайт и CRM
- apps/api — Fastify API
- apps/bot — Telegram bot/worker
- packages/db — база/Drizzle
- PostgreSQL
- PM2 процессы:
  - viberimenya-api-v2
  - viberimenya-web-v2
  - viberimenya-bot-v2

Рабочая папка:
- /var/www/viberimenya-v2

Домен:
- https://viberimenya.ru

## Важные правила
1. Не использовать патчи. Делать понятные изменения в файлах.
2. Перед изменением файла создавать backup этого файла.
3. Не трогать .env без отдельного разрешения.
4. Не показывать и не выводить секреты из .env.
5. Не менять Nginx, Basic Auth, SSL и firewall без отдельного разрешения.
6. Не делать DROP/TRUNCATE в базе.
7. Не удалять файлы массово.
8. Не запускать опасные команды типа rm -rf без объяснения и подтверждения.
9. Не менять одновременно несколько больших блоков.
10. После каждого изменения обязательно запускать проверки.

## Проверки
Для bot:
pnpm --filter @viberimenya/bot typecheck

Для api:
pnpm --filter @viberimenya/api typecheck

Для web:
pnpm --filter @viberimenya/web typecheck
pnpm --filter @viberimenya/web build

После успешных проверок перезапускать только нужный процесс:
pm2 restart viberimenya-bot-v2 --update-env
pm2 restart viberimenya-api-v2 --update-env
pm2 restart viberimenya-web-v2 --update-env

После перезапуска проверять:
pm2 status
pm2 logs <process-name> --lines 30 --nostream
